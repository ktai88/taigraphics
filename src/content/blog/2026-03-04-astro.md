<section><div class="page-container-800"><h2 class="text-2xl">Cut the fluff; Astro.Build als framework</h2><p class="leading-8 mt-3">Wist je dat een gemiddelde website met een standaard CMS (vb. Wordpress) 500MB tot 1GB diskspace nodig heeft, voor een basis website? Heb je die dan ook echt nodig voor je site? Ik wilde een framework dat van nature schoon en minimaal is. Geen CMS met eindeloze updates, plugins en performance-puzzels, maar een oplossing die vanaf dag 1 efficientie vooropstelt.</p><p class="leading-8 mt-3">Astro.build biedt precies dat: een static site builder die HTML, CSS en JavaScript genereert; zonder database en onnodige serverprocessing. Breng je project terug naar een behapbare set-up. Resultaat? Een portfolio van minder dan 4MB (exclusief assets), sub-second laadtijden met minimale carbon footprint.</p><img src="/astrobuild.png" class="w-full my-4 rounded-lg shadow" alt="Persona voorbeeld"/><p><em>Unique Selling Point van: Astro.Build</em></p><div class="grid grid-cols-[100%] gap-2 card-base card-stack"><div class="p-6"><h3 class="text-xl pt-1 pb-2 mb-2 inline-block border-bottom-gradient">Tip van Tai, stappenplan:</h3><p class="leading-8"><em>Welke stappen heb ik ondernomen:</em></p><ol class="list-disc pl-6 mt-3 text-[color:var(--color-text-muted)] text-sm"><li type="1" class="pb-4">Onderzocht welke framework en tools het beste past bij mijn doel: een lightweight, snel en duurzaam portfolio. Daarnaast gaf het mij veel controle op mijn eigen HTML structuur.</li><li type="1" class="pb-4">Ik koos voor een leeg nieuw project met Astro. Maar er bestaan ook starter-templates. In dit geval gebruikte ik TailwindCSS als startpunt (standaard spacings etc.). En heb ik mijn Figma-designs erop gebaseerd met dezelfde spacing en met een eigen thema en design.</li><li type="1" class="pb-4">Integreer TailwindCSS in je Astro.Build project voor het gebruik van utility classes en styling, dit zorgt ook voor een consistente design in je project. Maak losse componenten voor consistentie en hergebruik.</li><li type="1" class="pb-4">Project lokaal testen en verifieren. Via Visual Studio Code project inrichten en installeren van de Astro Plugin. En vervolgens mijn eerste projectstructuur met Astro.Build naar GitHub gepushed voor versiebeheer.</li><li type="1" class="pb-4">Kies een hosting provider en zet je project live. Ik koos voor een kleine en lightweight VPS met Ubuntu, maar er zijn ook andere opties zoals Netlify of Vercel waarop Astro.Build ook op kan draaien.</li><li type="1" class="pb-4">Gebruik gemaakt van Mistral Le Chat voor de verdere inrichting en optimalisatie van het project en VPS, en uiteraard de koppeling met SSL, Domein en VPS. De kennis voor de voorgaande stappen had ik al, maar ook hier voor kun je Le Chat gebruiken om je te helpen met inrichten van je project!</li></ol></div></div><p class="leading-8 content-block"><a href="/blog" class="mb-2 py-2 px-6 inline-block btn-primary">Terug naar overzicht</a></p></div></section>
